## 手動測試計劃

### 第 1 級：基本功能（推薦立即測試）

> 💡 本地運行 `npm run dev` 後，訪問 http://localhost:3000

#### 1. FlashCard 翻轉測試
```
步驟:
1. 打開首頁
2. 在卡片上點擊（或按 Enter/Space）
3. 觀察卡片以 0.6 秒動畫翻轉
4. 應顯示：POS 標籤、例句、詞形變化
5. 再次點擊翻回問題面

預期結果: 卡片平滑翻轉，內容正確
所需時間: 1 分鐘
```

#### 2. 語音功能測試
```
步驟:
1. 保留卡片在答案面
2. 點擊「朗讀 apple」按鈕
3. 應聽到 "apple" 的發音
4. 點擊「朗讀 She ate...」
5. 應聽到完整例句的發音

預期結果: 所有語音按鈕都能朗讀
所需時間: 2 分鐘
設備: 有揚聲器或耳機的計算機
```

#### 3. 測驗流程測試
```
步驟:
1. 選擇任意答案（對或錯）
2. 觀察選中的選項顏色
3. 點擊「下一題」進入下一個問題
4. 重複直到所有問題完成
5. 查看結果屏幕上的得分

預期結果: 
- 正確答案變綠
- 錯誤答案變紅
- 進度條逐步增加
- 最終顯示分數統計

所需時間: 3-5 分鐘
```

#### 4. 響應式設計測試
```
步驟 A (桌面):
1. 在瀏覽器中打開應用
2. 調整窗口為 1280x800（或更寬）
3. 驗證佈局和字體大小合適

步驟 B (平板):
1. 使用開發者工具切換到「平板」視圖 (768x1024)
2. 驗證按鈕和卡片可以正常交互

步驟 C (手機):
1. 使用開發者工具切換到「移動」視圖 (375x812)
2. 驗證文本可讀，按鈕可點擊

預期結果: 在所有尺寸下都能正常使用
所需時間: 2 分鐘
```

### 第 2 級：高級功能（部署後測試）

#### 5. Excel 匯入測試
```
前置條件: 已設置 Supabase 和部署

步驟:
1. 登錄應用
2. 進入卡組的匯入頁面 (/decks/[id]/import)
3. 點擊「下載範本」並打開 Excel
4. 編輯模板：
   - 第 1 行: 設置正確的值
   - 第 2 行: 只填 "word"（缺少 correct_meaning）
   - 第 3 行: 留空（測試空行）
5. 上傳修改後的文件
6. 驗證:
   - 第 1 行顯示「✓ 正常」
   - 第 2 行以紅色突出顯示
   - 顯示「缺少 correct_meaning」錯誤

預期結果: 驗證正常工作，錯誤行被突出顯示
所需時間: 3 分鐘
```

#### 6. RLS 隔離測試
```
前置條件: 有 2 個 Supabase 帳戶

步驟:
1. 用戶 A: 創建卡組「我的卡組」
2. 用戶 A: 添加幾個單字
3. 用戶 B: 登錄，試圖直接訪問卡組 URL
4. 用戶 B: 應看到 404 或無權限提示

預期結果: 用戶 B 無法訪問用戶 A 的卡組
所需時間: 5 分鐘
```

#### 7. 自動朗讀測試

## 已知限制

### iOS Safari
- 自動朗讀被禁用（需要用戶手勢）
- 解決方案: 用戶可以手動點擊朗讀按鈕

### 低速網絡
- 大型 Excel 文件可能需要更長時間上傳
- 建議: 限制單個檔案大小 < 5MB

### 瀏覽器兼容性
- 不支持 IE 11（使用現代瀏覽器）
- Web Speech API 在某些設備上不可用
